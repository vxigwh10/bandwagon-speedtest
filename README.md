## 三种测速方法，从简单到深入

### 方法一：本地 Ping 测延迟（最快，2 分钟搞定）

这是最基础的测法，测出来的是你本地到目标机房的往返延迟（RTT）。

**Windows 用户：**
1. 按 `Win + R`，输入 `cmd` 回车
2. 输入命令：`ping 65.49.131.102`（DC9 机房为例）
3. 看最后一行的平均时延，单位 ms

**Mac / Linux 用户：**
打开终端，直接输入 `ping 65.49.131.102`，看平均 ms 值。

参考延迟范围（国内用户，仅供参考，实际值因运营商和地区而异）：
- 香港机房（93.179.124.115）：广东方向约 10-30ms，华北约 40-60ms
- DC9 CN2 GIA（65.49.131.102）：国内约 150-180ms
- DC6 CN2 GIA-E（162.244.241.102）：国内约 150-190ms
- 日本东京 DC39（74.82.192.227）：国内约 80-120ms

延迟在 150ms 以内，建站体验不会太差。晚高峰能保持在 200ms 以内且丢包率低，就算稳定线路。

### 方法二：演示站点 SpeedTest（看实际下载速度）

直接打开演示站点，里面有 SpeedTest 工具，点一下"开始测试"，就能看到下载速度、上传速度、延迟波动。

比单纯 ping 更直观，还能看出带宽有没有被限。

- DC9 演示站：dc9.bwg.net
- DC6 演示站：dc6.bwg.net
- 日本东京 DC39：dc39.bwg.net
- 日本大阪软银 JPOS_1：jpos.bwg.net
- 美国弗里蒙特：fmt.bwg.net

打开演示站点，点进 SpeedTest 页面，然后点开始测试，10 秒左右出结果。下载速度能跑到几十 Mbps 以上的机房，日常使用问题不大。

### 方法三：LookingGlass 路由追踪（看你的流量走哪条路）

演示站点里除了 SpeedTest，还有 LookingGlass 功能，支持 ping、traceroute、MTR 等测试。

这个方法最关键——延迟好看不代表线路好，关键是看流量走的是 CN2 GIA 还是普通 163 线路。晚高峰的体验，90% 取决于线路质量，而不是延迟数字本身。

测试步骤：
1. 打开对应机房的演示站点（比如 dc9.bwg.net）
2. 进入 LookingGlass 页面
3. 在输入框里填你本地的 IP（或者你的服务器 IP）
4. 选择 traceroute，点开始测试
5. 看结果里有没有出现 `59.43.x.x`（这是电信 CN2 骨干），出现了说明走的是 CN2 GIA

搬瓦工电信用在 CN2 GIA 机房上，每一跳出现 `59.43` 开头的 IP，就是在走精品网段，晚高峰基本不卡。

## 搬瓦工测速 FAQ

**Q：搬瓦工官方测速 IP 和第三方测速 IP 有什么区别？**

搬瓦工官方提供的测速 IP 直接架设在其自有数据中心，测出的延迟和路由是最贴近实际 VPS 网络表现的数据。第三方 IP 可能不在同一机房，参考价值有限。

**Q：ping 延迟低，但实际用起来还是慢，是什么原因？**

延迟低不等于速度快。真正影响使用体验的是线路质量和丢包率，尤其是晚高峰时段的表现。建议用 LookingGlass 做路由追踪，确认流量是否走 CN2 GIA 线路，同时在晚高峰（晚上 8-11 点）再测一次，对比白天数据。

**Q：没有机器，怎么测搬瓦工的路由？**

用搬瓦工各机房的演示站点（LookingGlass 功能），输入你本地 IP，让搬瓦工的服务器从那边对你发包做路由追踪，就能看到从机房出来到你本地的回程路由。

**Q：测速 IP 可以直接用来购买 VPS 吗？**

不行。测速 IP 只是用于网络测试，不是 VPS 的 IP，不能当服务器用。购买 VPS 要走官网套餐下单流程。

**Q：搬瓦工哪个机房对国内用户最快？**

延迟最低的是香港机房，广东方向在 10-30ms 之间。性价比最高的是 DC6 CN2 GIA-E 或 DC9 CN2 GIA，三网 CN2 GIA 线路，晚高峰稳定性好，价格比香港套餐低很多。
